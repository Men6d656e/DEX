📄 Smart Contracts

Three contracts form the backend of the DEX Dashboard: MockERC20 (token), Faucet (distribution), and MockDEX (trading).

MockERC20

OpenZeppelin-based ERC20 token with mint capability restricted to a designated faucet address.

Contract: MockERC20
Inherits: OpenZeppelin ERC20, Ownable
Key Features:
- constructor(name, symbol, faucetAddress)
- mint(to, amount) — only callable by the faucet address
- setFaucet(newFaucet) — onlyOwner
- TokensMinted(address indexed to, uint256 amount) event

Deployment: Three instances — mETH, mBTC, and mUSDC.

Faucet

Time-locked token distribution contract with per-user, per-token 24-hour cooldown tracking.

Contract: Faucet
Inherits: OpenZeppelin Ownable
State:
- lastClaim[user][token] → uint256 (unix timestamp)
- lifetimeClaimed[user][token] → uint256
- COOLDOWN: 24 hours
- CLAIM_AMOUNT: 10 tokens (with 18 decimals)
Key Functions:
- claimToken(tokenIndex) — 0 = mETH, 1 = mBTC
- getClaimInfo(user, tokenIndex) → (canClaim, timeRemaining, lifetimeClaimed, lastClaimTime)
- setClaimAmount(amount) — onlyOwner
- setCooldown(seconds) — onlyOwner
Errors:
- Faucet__CooldownNotElapsed(remaining)
- Faucet__InvalidToken()

MockDEX

Simple ratio-based swap contract enabling mETH ↔ mUSDC trades with slippage protection.

Contract: MockDEX
Inherits: OpenZeppelin Ownable
State:
- mETH, mUSDC: IERC20 token addresses
- ethReserve, usdcReserve: uint256
- swapRate: uint256 (e.g., 1700 = 1 mETH = 1700 mUSDC)
Key Functions:
- addLiquidity(ethAmount, usdcAmount) — onlyOwner
- swapETHForUSDC(ethAmount, minUSDC)
- swapUSDCForETH(usdcAmount, minETH)
- getRate() → uint256
- setRate(newRate) — onlyOwner
Errors:
- MockDEX__InsufficientLiquidity()
- MockDEX__SlippageExceeded()
- MockDEX__ZeroAmount()

Testing

All contracts use Foundry's forge test with Solidity-based unit tests, fuzz testing, and custom error assertions. Target is 100% line coverage. See the Testing Guide for details.
