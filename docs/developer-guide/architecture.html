🏗 System Architecture

Monorepo Structure

The project is organized as a monorepo with three main directories:

DEX/
├── contracts/         # Foundry (Solidity) project
│   ├── src/           # Smart contracts
│   ├── test/          # Solidity tests
│   └── script/        # Deployment scripts
├── frontend/          # Next.js 16 application
│   ├── src/
│   │   ├── app/       # Pages (Home, Faucet, Swap, Analytics)
│   │   ├── components/# UI components
│   │   ├── hooks/     # Custom React hooks
│   │   └── lib/       # Utilities & constants
├── docs/              # GitHub Pages documentation
├── Makefile           # Build automation
└── SPEC.md            # Architecture specification

Data Flow

  1. User connects wallet via wagmi + MetaMask connector
  2. Frontend reads on-chain data using wagmi hooks (useReadContract, useReadContracts)
  3. User submits transactions through wagmi's useWriteContract
  4. Chart data is fetched from CoinGecko API with simulated fallback
  5. Analytics data combines on-chain reserves with simulated trade data

Key Design Decisions

  • Foundry — Native Solidity testing with fuzz support, fast compilation
  • wagmi + viem — Modern React hooks for wallet connection and contract interaction
  • shadcn/ui — Accessible, customizable Radix UI primitives with Tailwind
  • Ratio-based swap — Educational simplicity over AMM complexity
  • Raw HTML docs — GitHub Pages compatible with zero build step
